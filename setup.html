Setup & prerequisites

Running the reference, start to finish.

From a bare Fedora workstation to a fully running data mesh — kernel tuning, tooling, and a single bootstrap command that brings up the entire stack: Istio, Kafka, Postgres, OpenMetadata, observability, and every domain service.

Verified configuration

Requirement Minimum Notes
OS Fedora 44 Rootless podman as container runtime
RAM 64 GB total 24 GB allocated to minikube profile; rest is host headroom
Disk 1 TB 30 GB free minimum for image cache + PVs
CPU 16 vCPU recommended Not strictly required but the stack is heavy

Kernel & container tuning

Legacy iptables modules

Fedora is nftables-only; the rootless minikube node can't modprobe itself. Without these modules CNI portmap fails. 

sudo sh -c 'printf "ip_tables\niptable_nat\nip6_tables\n" \
    > /etc/modules-load.d/99-kubernetes-iptables.conf'
sudo systemctl restart systemd-modules-load

inotify limits

The capstone runs many controllers; Fedora's default 128 instances is not enough. 

sudo sh -c 'printf "fs.inotify.max_user_instances = 512\nfs.inotify.max_user_watches = 524288\n" \
    > /etc/sysctl.d/99-kubernetes.conf'
sudo sysctl -p /etc/sysctl.d/99-kubernetes.conf

Podman pids_limit

The fully-meshed stack runs ~2000+ tasks; Fedora's default 2048 saturates. Set this before creating the minikube profile. 

mkdir -p ~/.config/containers
printf '[containers]\npids_limit = 0\n' >> ~/.config/containers/containers.conf

Required tooling

  • minikube >= 1.36 — 1.35's registry addon pins a kube-registry-proxy image digest that no longer exists on gcr.io
  • kubectl
  • helm
  • istioctl — with the full Istio distribution (not just the binary; setup-kiali.sh applies samples/addons/kiali.yaml from it)

Installing the Istio distribution

curl -fsSL https://github.com/istio/istio/releases/download/1.26.2/istio-1.26.2-linux-amd64.tar.gz \
    | tar xz -C ~/.local/share
ln -sfn ~/.local/share/istio-1.26.2 ~/.local/share/istio-current
cp ~/.local/share/istio-current/bin/istioctl ~/.local/bin/

Bootstrap

Bring the whole system up

cd examples/lgtm-datamesh
./scripts/bootstrap-capstone.sh

Bootstrap runs 10 tiers: minikube profile + registry, Istio, CloudNativePG operator, Postgres cluster, Kafka (Strimzi), KEDA + HTTP add-on, OpenMetadata + observability, all services + scalers + seed data, Kiali, and catalog ingestion + lineage.

Restore baseline after running demos

./scripts/restore-baseline.sh

The demo scripts clean up their own releases on success. After running several, shared services may be missing. restore-baseline.sh puts the workloads back in their bootstrap state.

Verifying the stack

Cluster status

./scripts/cluster-status.sh

Quick verification demos

  1. ./demos/demo-order.sh — creates an order via REST, confirms it in Postgres
  2. ./demos/demo-service.sh inventory — builds, deploys, and health-checks a service
  3. ./demos/demo-observability.sh — verifies Prometheus is scraping and Grafana is provisioned

Accessing the observability tools

The tools run inside the cluster. Port-forward to reach them from the host — Fedora ships Cockpit on port 9090, so Prometheus uses an alternate local port.

Tool Port-forward command Local URL
Grafana kubectl port-forward -n capstone svc/grafana 3000:80 http://localhost:3000
Prometheus kubectl port-forward -n capstone svc/prometheus-server 9091:80 http://localhost:9091
Kiali kubectl port-forward -n istio-system svc/kiali 20001:20001 http://localhost:20001/kiali
Tempo (via Grafana) — (use Grafana → Explore → Tempo datasource) —
OpenMetadata kubectl port-forward -n capstone svc/openmetadata 8585:8585 http://localhost:8585

Grafana default credentials are admin / the value from kubectl get secret grafana -n capstone -o jsonpath='{.data.admin-password}' | base64 -d.
