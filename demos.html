Demos & examples

See every principle, live.

23 scripts under examples/lgtm-datamesh/demos/ exercise the data-mesh reference end to end. Each one is mapped to the principle it demonstrates — domain ownership, data as a product, self-serve platform, or federated governance.

The five-act walkthrough

walkthrough.sh orchestrates five acts — trace, scale, canary, lineage, topology — each shelling out to an existing demo script with narration between them. Press Enter to advance between acts. 

cd examples/lgtm-datamesh && ./demos/walkthrough.sh

Running a demo

All demos assume you are in the examples/lgtm-datamesh/ directory with kubectl context set to capstone. See the setup & prerequisites page for installation and bootstrap. 

cd examples/lgtm-datamesh
./demos/demo-order.sh

What to look at after running a demo

  • Grafana — dashboards for request rate, canary weight, KEDA replicas:
    kubectl port-forward -n capstone svc/grafana 3000:80 → http://localhost:3000
  • Kiali — live mesh topology graph:
    kubectl port-forward -n istio-system svc/kiali 20001:20001 → http://localhost:20001/kiali
  • Tempo — distributed traces: open Grafana → Explore → select the Tempo datasource
  • OpenMetadata — catalog and lineage:
    kubectl port-forward -n capstone svc/openmetadata 8585:8585 → http://localhost:8585

Restoring baseline

The demo scripts clean up their own releases on success. After running several, shared services may be missing. To put the workloads back in bootstrap state: 

./scripts/restore-baseline.sh
